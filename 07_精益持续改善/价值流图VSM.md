# 🗺️ 价值流图（VSM）

> Value Stream Mapping — 从客户到供应商的端到端可视化

---

## 什么是VSM

VSM是用铅笔和纸画出从原材料到客户手中所有流程（含信息流和物流），识别浪费、找到改善机会的精益工具。

## VSM的核心要素

### 两个流向
```
信息流（右→左）：客户订单 → 生产计划 → 采购指令 → 供应商
物料流（左→右）：来料 → 加工 → 组装 → 检验 → 发货 → 客户
```

### 关键符号

| 符号 | 含义 |
|------|------|
| ▷▷ | 推动（Push） |
| △ | 超市/库存缓冲 |
| ◇ | 看板信号 |
| □ | 工序/流程框 |
| ⚡ | 改善爆发点（Kaizen Burst） |
| 📦 | 库存三角形 |
| ⏱️ | 时间线（C/T, C/O, Uptime） |

## VSM绘制九步法

### 1. 选定产品族
选择占产值60-80%的产品或产品族

### 2. 绘制现状图
**客户端：**
- 客户月/周需求量
- 包装方式、发货频次

**流程框：**
- 每个工序画一个框
- 标注：C/T（节拍）、C/O（换型时间）、Uptime（开动率）
- 操作人数

**库存三角：**
- 各工序间的在制品数量
- 原材料和成品库存

**信息流：**
- 订单传递方式
- 排程方式（日/周/月）

**时间线：**
- 增值时间（加工时间）
- 非增值时间（等待/搬运/库存）
- 计算：价值流效率 = 增值时间 / 总周期时间

### 3. 识别浪费
- 过量生产、等待、搬运、加工、库存、动作、不良
- 标记改善爆发点

### 4. 设定目标
- 目标节拍时间、目标WIP、目标周期时间

### 5. 设计未来状态图
按照精益原则设计理想流程：
- 按客户节拍生产
- 连续流（Continuous Flow）尽可能延伸
- 无法连续流的地方设超市拉动
- 排程点只有一个（定拍工序）
- 均衡化生产（Heijunka）
- 缩短换型时间

### 6. 制定改善计划
从现状到未来的具体行动：
- 立即可以做的
- 本周要完成的
- 本月要实现的
- 年度目标

### 7. 执行改善
- 组建改善团队
- PDCA循环推进

### 8. 确认效果
- 对比改善前后VSM
- 量化成果（周期缩短XX%、库存减少XX%）

### 9. 标准化与持续
- 新标准融入日常管理
- 定期更新VSM（季度/半年）

## VSM分析的关键指标

| 指标 | 公式 | 行业参考 |
|------|------|---------|
| 增值比 | 增值时间/总周期时间 | 精益<10%即可，多数行业<5% |
| 节拍时间 | 可用时间/需求数量 | 匹配客户需求 |
| 周期时间 | 原材料到成品时间 | 越短越好 |
| 加工时间 | 各工序加工时间总和 | 增值部分 |
| 在制品周转 | 总库存/日均产出 | 日消耗量为佳 |

## VSM实战案例

### 改善前
```
客户需求：100件/天
原料 → 冲压(1000件/WIP 3天) → 焊接(500件/WIP 2天) → 组装(200件/WIP 1天) → 成品(500件/WIP 3天)
增值时间：180秒  
总周期时间：9天
价值流效率：0.02%
```

### 改善后
```
原料 → 冲压(100件/WIP 5hr) → 焊接(100件/WIP 3hr) → 组装(50件/WIP 1hr) → 成品(200件/WIP 1天)
增值时间：180秒
总周期时间：1.5天
价值流效率：0.14% (提升7倍)
```

## VSM常见坑

| 坑 | 建议 |
|------|------|
| 画得太完美 | 现状图必须如实反映，即使很糟糕 |
| 只画物料流 | 信息流同样重要，甚至更重要 |
| 没有数据支撑 | 每个工序的C/T、C/O必须实际测量 |
| 一次性想要完美 | 先画好现状图，再逐步改善 |
| 改善后不更新 | VSM应是动态的，定期复盘 |
