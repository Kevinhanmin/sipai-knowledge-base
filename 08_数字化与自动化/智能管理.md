# 📊 智能管理

> 将生产现场数据转化为管理层的可决策信息——数据驱动的管理闭环

---

## 一、ERP演进与云端选项

### 传统ERP的局限
- 流程僵化，难以适配柔性生产需求
- 数据孤岛，与MES/WMS集成困难
- 批处理模式（T+1）无法满足实时管理

### 云ERP选项

| 平台 | 特点 | 适合 |
|------|------|------|
| SAP S/4HANA Cloud | 内存计算、实时分析 | 大型离散制造 |
| Oracle NetSuite | 全球化多实体支持 | 跨国制造 |
| 用友U8 Cloud | 本地化合规、性价比高 | 国内中小制造 |
| 金蝶云·星空 | 同上 | 同上 |
| Odoo | 开源模块化 | 初创、低成本 |

**思派建议**：中小企业优先国产云ERP（用友/金蝶），在数据安全与灵活性之间取得平衡。

## 二、BI商业智能与看板

### BI工具选型

| 类型 | 代表 | 适合 |
|------|------|------|
| 商业BI | Power BI / Tableau / 帆软 | IT资源有限的企业 |
| 开源BI | Superset / Metabase | 轻量自建 |
| 自建BI | Python+Streamlit/Grafana | 极客团队 |

### 智能工厂典型BI看板

1. **生产指挥屏**：实时OEE、产线节拍、WIP
2. **质量仪表盘**：SPC控制图、不良率趋势、FPY
3. **供应链驾驶舱**：OTIF、安全库存预警
4. **财务经营屏**：制造成本分析、库存周转

> **设计原则**：3-5秒法则——管理层在3-5秒内应抓住核心异常

## 三、管理驾驶舱设计

### 层次化设计

| 层级 | 适用角色 | 视角 | 指标数 |
|------|---------|------|:----:|
| 战略层 | 总经理 | 月/季度 | 5±2 |
| 经营层 | 厂长/运营总监 | 周/日 | 7±2 |
| 执行层 | 班组长 | 实时 | 5±2 |

### 视觉设计原则

- 左上角放最重要的KPI
- 红色=严重异常（需立即响应），黄色=预警，绿色=正常
- 图表类型：KPI卡片→折线趋势→热力图分布
- 一张屏不超过7个关键指标（米勒定律）

## 四、数据中台（精简版）

```text
数据源层 → ETL → ODS(原始) → DWD(明细) → DWS(汇总) → 指标中台 → BI/看板
  ERP                                                          API
  MES                                                        AI应用
  SCADA
  WMS
```

**核心原则**：能不建就不建，能用轻量工具（ClickHouse/Doris）就不上重平台。

## 五、KPI自动化

| 维度 | KPI | 计算公式 | 数据源 |
|:----:|:---:|:---------|:------:|
| 效率 | OEE | 可用率×性能率×良率 | MES+设备 |
| 质量 | FPY | 一次通过数/总生产数 | MES |
| 交期 | OTIF | 准时交付单数/总单数 | ERP |
| 成本 | 单位成本 | 总制造成本/合格产品数 | ERP+MES |
| 库存 | 周转天数 | 平均库存/(COGS/365) | ERP |

**推荐方案**：时序数据库(TDengine) + Python定时计算 + BI推送。避免用Excel手动搬运数据。

## 六、数据治理基础

### 五大领域

1. **数据标准**：统一物料编码、工序编号、设备ID
2. **数据质量**：完整性/准确性/一致性/及时性
3. **数据安全**：权限分级管理
4. **数据生命周期**：热(7天)→温(1年)→冷(归档)
5. **数据组织**：IT+业务各出1人成立数据治理小组

## 七、思派视角

### 从"经验驱动"到"数据驱动"

真正的转变需要：
- 仪表盘要成为**早会的规定动作**
- 指标要**可归因**——异常了知道去看哪个环节
- 数据要**可行动**——发现问题后系统能推荐调整方案

### 警惕"数据仪表盘装饰品"

> 一些企业花几十万做了一大屏，领导来参观时展示，平时没人看。**管理驾驶舱的灵魂是"用起来"，而不是"看起来"**。

### 五个原则

1. 先简后繁：先做月度KPI看板，再做实时看板
2. 指标越少越好：一张屏不超过7个关键指标
3. 数据干净比工具重要：脏数据进BI就是"漂亮垃圾"
4. 自动化维护：ETL管道要有监控告警
5. 管理层数字素养：再好的系统也架不住不看数据的领导
