# 📰 每日行业快讯 · 内容师工作流SOP

## 工作流概览

```
08:00 ─ 自动生成模板+飞书通知 → 09:00 ─ 内容师填充 → 14:00 ─ 诊断师审核
                                        ↓                          ↓
                                 内容师生成初稿              审核通过 → 推送创始人飞书
                                                               ↓
                                                         不通过 → 内容师修改
```

## 自动化管线

### 每日 08:00 — LaunchAgent 自动执行

系统通过 LaunchAgent (`com.sipai.daily-newsletter`) 每日 08:00 自动运行：

```bash
python3 ~/Documents/精益智能工厂/02_生产管理/每日快讯/daily_newsletter_full.py
```

**自动执行的内容：**
1. ✅ 生成当日快讯空白模板 `每日快讯_YYYYMMDD.md`
2. ✅ 通过飞书消息通知创始人：「今日快讯模板已就绪，请等待内容师填充+诊断师审核」

### 审核通过后 — 推送创始人微信

诊断师审核通过后，由诊断师或Orion调用推送脚本：

```bash
# 推送今日快讯
~/Documents/精益智能工厂/02_生产管理/每日快讯/push_newsletter_to_wechat.sh

# 推送指定日期快讯
~/Documents/精益智能工厂/02_生产管理/每日快讯/push_newsletter_to_wechat.sh 2026-05-12
```

推送内容：快讯摘要（政策动态/行业案例/工具方法/趋势洞察/老K点评）
推送渠道：飞书消息 → 创始人微信（飞书已绑定微信）

## 角色分工

| 角色 | 职责 | 产出 |
|:----:|------|------|
| **内容师** | 收集行业资讯、填充模板、撰写点评 | 初稿快讯（status: 待审核） |
| **诊断师** | 审核技术准确性（术语、数据、边界） | 审核结论（通过/需修改） |
| **创始人** | 最终发布确认 | 发布 |
| **Orion** | 编排执行、监控进度 | 无直接产出 |

## 每日执行步骤

### Step 1：检查模板（08:00）

每天早上 08:00，系统自动生成当日快讯模板：
```
~/Documents/精益智能工厂/02_生产管理/每日快讯/每日快讯_YYYYMMDD.md
```

内容师检查模板是否正常生成。

### Step 2：内容师收集资讯（08:30-09:00）

使用 web_search 搜索以下4个方向：

**方向1：政策动态**
- 搜索词：「制造业 技改补贴 2026」「数字化转型 中小企业 政策2026」
- 关注：工信部、各省经信厅官网通知
- 筛选：与中小企业、传统制造业相关的政策

**方向2：行业案例**
- 搜索词：「精益生产 成功案例 工厂 2026」「智能工厂 数字化转型 案例 2026」
- 关注：有具体数据（效率提升%、成本降低%）的案例
- 筛选：与IP定位（精益智能工厂实战派）一致的案例

**方向3：工具方法**
- 搜索词：「TPM OEE 提升 技巧」「5S 现场管理 精益工具」
- 关注：可实操、中小工厂能用的方法
- 注意：术语必须准确，后续需诊断师审核

**方向4：趋势洞察**
- 搜索词：「制造业 趋势 2026」「AI 制造业 应用」
- 关注：有数据支撑的趋势分析
- 避免：纯理论、无数据支撑的预测

### Step 3：填充模板（09:00-09:30）

每节填充2条资讯，格式统一：
```markdown
### [具体标题]
- **要点**：一句话总结
- **影响**：对中小制造企业的实际影响
- **来源**：[URL]
```

### Step 4：添加老K点评（09:30）

结合创始人IP人设撰写点评：
- 样式：「老K说：……」「在比亚迪时我们遇到过类似情况……」
- 定位：实战、接地气、有观点
- 长度：50-100字

### Step 5：提交审核（09:30）

更新status为「待审核」，通知诊断师：
- 使用 `mention://agent/22f43346-9ec4-4eb4-b481-abf8b1b5d2cf` 通知诊断师

### Step 6：诊断师审核（14:00前）

诊断师检查：
- ✅ 技术术语是否正确（OEE/TPM/SMED）
- ✅ 数据是否合理、有来源
- ✅ 是否超出免费内容边界
- ✅ 是否与IP人设一致

### Step 7：发布（审核通过后）

- 公众号：排版后发布（周日可用深度图文）
- 视频号：提取关键点制作短视频
- 微信群：摘要发到客户群

## 内容边界规则

- ✅ **可以写**：政策解读、行业趋势、工具科普、成功案例概述
- ❌ **不能写**：客户具体数据、收费诊断内容、未经证实的行业数据
- ⚠️ **需谨慎**：涉及具体企业的案例（需脱敏处理）

## 快讯质量标准

| 维度 | 标准 |
|:----:|------|
| 时效性 | 信息不超过1周 |
| 相关性 | 与精益/智能工厂直接相关 |
| 准确性 | 术语正确、数据有来源 |
| 可读性 | 每节不超过150字，让工厂老板3分钟读完 |
| 价值性 | 每条资讯有明确「可借鉴点」 |

## 异常处理

| 场景 | 处理方式 |
|:----:|---------|
| 当天无足够新闻 | 用知识文库已有内容填充，标注「内容回顾」 |
| 政策信息复杂 | 只摘核心要点，详细解读放入周日长文 |
| 疑似数据造假 | 标注「未经证实」，不过度解读 |
| 诊断师审核不通过 | 修改后重新提交，2小时内完成 |
